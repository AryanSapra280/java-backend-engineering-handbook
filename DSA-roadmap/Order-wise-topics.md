# Pattern Recognition Checklist

Before writing code, always answer the following:

* What is the input?
* What is the output?
* Is the array sorted?
* Is this asking for a pair?
* Is this a contiguous subarray?
* Is this a substring?
* Is it asking for the longest or shortest?
* Is there a frequency constraint?
* Is this a graph or dependency problem?
* Does recursion naturally fit?
* Can overlapping subproblems be observed?

Only after answering these questions should coding begin.

# Rule of Mastery

A pattern is considered mastered only when:

* You can identify it within 30 seconds.
* You can explain why other patterns do not fit.
* You can write the solution without looking at notes.
* You can solve at least three variations.
* You can explain the solution to another engineer.
